enum ConfigRW.Parsing.Lexer.state
private

Possible states of processing

Enumerator
gatherIdentifierStart 

Next step is expected to gather identifier start, space may occur, in which case, state is not changed

gatherIdentifierStart_escaped 

first non-space character was escaper, so identifier start is yet to come

gatherIdentifierBody 

Identifier start is gathered, next step is expected to gather rest of the body

gatherIdentifierBody_spaces 

Identifier start is gathered, next step is expected to gather rest of the body, last gahered characters were spaces, possiblility of space rollback

gatherIdentifierBody_escaped 

Identifier start is gathered, next step is expected to gather rest of the body, escaper was present in the last step

newLexemeStart 

Another lexeme in value part of option is expected, if spaces occur, they should be ignored, if comment section start occurs, comment state is expected

newLexemeStart_ignoreDelimiter 

Another lexeme in value part of option is expected, if spaces occur, they should be ignored, if comment section start occurs, comment state is expected This state is special, that previous lexeme definitely ended, howerver nothing is known about presence of next delimiter, it may or may not be present

gatherElementBody 

In progress of gathering element body

gatherElementBody_escaped 

In progress of gathering element body, last character was escaper

gatherElementBody_spaces 

In progress of gathering element body, collecting spaces, possibility of spaces rollback

gatherLinkSection 

Gatherink link section

gatherLinkSection_escaped 

Gathering link section, last character was escaper

gatherLinkOption 

Gathering link option

gatherLinkOption_escaped 

Gathering link option, last character was escaper

comment 

Comment area occurred, next of the line is just the comment
